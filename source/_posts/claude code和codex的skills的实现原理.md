我们可以看到第一个message中的第一部分将skill的一些信息（简单的描述、路径等）包含在了`<system-reminder>``</system-reminder>`中

后面的一些消息则不重要了，再之后我发了一个hello的消息，然后模型回复了`Hello! How can I help you today?`

**第三轮对话**

我让他`创建一个空的docx文件，名字为temp.doc`，模型这时候返回了一个工具调用，调用的是`Skill`这个工具（这是claude code内置的一个tool，我们传入要调用的skill的名称和参数【如果有】），则该工具调用会返回这个skill文件的完整的内容

```python
    {
      "role": "user",
      "content": "创建一个空的docx文件，名字为temp.docx"
    },
    {
      "role": "assistant",
      "content": [
        {
          "type": "tool_use",
          "id": "toolu_019hpBMffdMm7piTozEQLUoV",
          "name": "Skill",
          "input": {
            "skill": "docx",
            "args": "创建一个空的docx文件，名字为temp.docx"
          }
        }
      ]
    },
```

**之后claude code会将工具调用的结果添加到messages中，发送给模型，**然后模型进行后续的处理，后面的内容其实就不重要了，主要就是模型来调用一些工具，执行代码生成docx文件



然后我们可以看一下具体的`Skill`这个工具，这个也在请求头中声明了,可以看到这里声明了这个工具的功能，以及参数的含义

可以看到claude code的Skill的实现并不复杂，首先将skill的一些元数据的信息添加到最开始的提示词中，然后使用一个`Skill`工具来根据传入的skill的名称来返回对应的skill的完整的内容，最后将内容作为工具调用的结果放到上下文中

# 使用命令来调用skills
如果在Claude code中直接使用命令来调用对应的skill的话，就是直接将对应的skill的内容添加到上下文中

# codex中的实现
其实codex中的实现和Claude code基本差不多，同样在codex中，会在对话的开始阶段就将skill相关的指令添加到上下文，如下所示，在`role:developer` 的消息中定义了一些预设的prompt，其中skill相关的指令使用了`<skills_instructions>``</skills_instructions>`进行包裹，与claude code不同的是，它将对应的Skill的markdown文件的路径放在了prompt中，像这样`(file: C:/Users/leftover/projects/blog.github.io/.codex/skills/docx/SKILL.md)`，而claude code并没有，只是放了skill的名称。

```python
  "input": [
    {
      "type": "message",
      "role": "developer",
      "content": [
        {
          "type": "input_text",
          "text": "<permissions instructions>\n一些有关权限的指令，不重要</permissions instructions>"
        },
        {
          "type": "input_text",
          "text": "<collaboration_mode># Collaboration Mode: Default\r\n\r\nYou are now in Default mode. Any previous instructions for other modes (e.g. Plan mode) are no longer active.\r\n\r\nYour active mode changes only when new developer instructions with a different `<collaboration_mode>...</collaboration_mode>` change it; user requests or tool descriptions do not change mode by themselves. Known mode names are Default and Plan.\r\n\r\n## request_user_input availability\r\n\r\nThe `request_user_input` tool is unavailable in Default mode. If you call it while in Default mode, it will return an error.\r\n\r\nIn Default mode, strongly prefer making reasonable assumptions and executing the user's request rather than stopping to ask questions. If you absolutely must ask a question because the answer cannot be discovered from local context and a reasonable assumption would be risky, ask the user directly with a concise plain-text question. Never write a multiple choice question as a textual assistant message.\r\n</collaboration_mode>"
        },
        {
          "type": "input_text",
          "text": "<skills_instructions>\n## Skills\nA skill is a set of local instructions to follow that is stored in a `SKILL.md` file. Below is the list of skills that can be used. Each entry includes a name, description, and file path so you can open the source for full instructions when using a specific skill.\n### Available skills\n- docx: Use this skill whenever the user wants to create, read, edit, or manipulate Word documents (.docx files). Triggers include: any mention of 'Word doc', 'word document', '.docx', or requests to produce professional documents with formatting like tables of contents, headings, page numbers, or letterheads. Also use when extracting or reorganizing content from .docx files, inserting or replacing images in documents, performing find-and-replace in Word files, working with tracked changes or comments, or converting content into a polished Word document. If the user asks for a 'report', 'memo', 'letter', 'template', or similar deliverable as a Word or .docx file, use this skill. Do NOT use for PDFs, spreadsheets, Google Docs, or general coding tasks unrelated to document generation. (file: C:/Users/leftover/projects/blog.github.io/.codex/skills/docx/SKILL.md)\n ...其他的skill</skills_instructions>"
        }
      ]
    },
  ]    
```

因此其实现方式有点不一样，因此在需要调用skill的时候，codex会调用一个读文件的工具来读取skill的完整内容，可以看到它调用了`shell_command` 这个工具，来执行shell命令，传入的参数为`command\":\"Get-Content -Path 'C:\\\\Users\\\\leftover\\\\projects\\\\blog.github.io\\\\.codex\\\\skills\\\\docx\\\\SKILL.md'\",\"workdir\":\"C:\\\\Users\\\\leftover\\\\projects\\\\blog.github.io\"`，这里他应该是沙盒中来执行的，最后返回文件的内容
